Haven is a parking reservation application. It is served in two platforms: website for long-term parking and mobile app for short-term parking For this project, all design decisions are made after discussions with members. Within the group of 6 members, my main task was to create a wireframe and prototype for the website.

For the last deliverable, we had to create a final project documentation for Haven. This documentation is for our client, a fake company called Data Solution, as a result of their request for an parking application that utilizes their data.

See Documentation

Based on the basic functions of Haven, I created the sitemap.
Sitemap describes the basic layout and interconnectivity of the pages on both platforms

After several group meetings, we made low-fidelity design decisions which are used for wireframes.

First one is for main(home) page.

Second one is for searching page.

Improving on the wireframes, I created two different low-fidelity protypes for A/B testing.

Version A: Sign up with an account first

Version A: User with an account viewing

Version B Search for parking immediately

Version B Anonymous viewing

Based on the feedback from A/B testing, we concluded three main design features for web platform:
1. Use the ‘try it before you buy it’ model
2. Keep the search bar prominently featured
3. Include some information about our product above the fold

Using these features, I created the final refined version of prototype.
